There are some things that I would expect from a public api to use it in a project

  • A good API - What ever you think that is: give it to me, please
  • Someone "in charge" of the code - person will do, a group is better
  • A github, gitlab or bitbucket url - Source-Code or die()
  • A set of tests
  • A way to 'build' the module
  • The Unit is in CI and I can see the results - travis <3
  • installation via a package manger - npm, gem, composer, bower
  • Markdown Docs
  • Source-Code API Docs - Build of these included please
